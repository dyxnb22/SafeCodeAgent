## Goals
- [x] Add `src/safecode/agent/planner.py` with `DiffPlanner`, `DiffPlan`, and `DiffScopeResult`.
- [x] Predict likely touched files from task description text before patch generation.
- [x] Compare predicted scope to actual patch files after LLM response is parsed.
- [x] Surface a structured warning when the patch touches files outside the predicted scope.
- [x] Do not block valid patches solely because scope differs (warning-only, not a gate).
- [x] Attach `scope_result` to `EditResult` and include scope metadata in the audit event.
- [x] Show the scope warning in the CLI after the diff is printed (yellow text).
- [x] Keep behavior deterministic and keyless (no LLM call for prediction).
- [x] Use `Literal` type for `DiffScopeResult.status` (Pydantic-enforced).
- [x] Deduplicate repeated blocks for the same file before comparison.
- [x] Use ASCII punctuation only in warning strings.

## Requirements
- [x] Prediction is read-only and uses only the task text and an optional context hint.
- [x] `DiffScopeResult.status` takes one of four values: `no_prediction`, `match`, `within_scope`, `extra_files`.
- [x] A warning string is only populated on `extra_files` status.
- [x] Scope comparison uses exact POSIX path matching against `PatchProposal.blocks[*].file_path`.
- [x] Existing `edit` workflow (parse → validate → diff → checkpoint → apply → audit) is unchanged.

## Backward Compatibility
- `EditResult` gains an optional `scope_result: DiffScopeResult | None = None` field (default None).
- Existing tests that check `result.proposal`, `result.diff_text`, and `result.pending_patch_path` are unaffected.
- The patch is not blocked or modified by the scope check.

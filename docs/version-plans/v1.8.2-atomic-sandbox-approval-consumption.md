# v1.8.2: Atomic Sandbox Approval Consumption

## Status
Implemented.

## Base
- Depends on: `v1.8.1`
- Baseline context: `.claude/skills/current/SKILL.md`
- Runtime invariants: `.claude/skills/shared/core-runtime.md`
- New tag: `v1.8.2`

## Problem
v1.8.1 has a TOCTOU race condition in `execute_pending()`: two concurrent
calls can both pass preflight (`is_approved()` returns True), both execute
`ShellRunner.run()`, and only then call `consume()`. The approval should
prevent the second execution but doesn't under concurrency.

## Goals
- [x] Close the TOCTOU window between preflight approval check and
      `ShellRunner.run()` by atomically claiming the approval BEFORE
      execution.
- [x] If claim fails, fail closed — do NOT execute the command.
- [x] Blocked preflight still does not consume the approval.
- [x] Approval is consumed as soon as execution is attempted, even if the
      command exits non-zero.
- [x] Audit event `sandbox_execution_approval_claimed` records the claim.
- [x] Audit events must not leak env values or full command content.
- [x] Legacy approval files (missing `consumed` field) remain claimable once.
- [x] Backward-compatible with v1.8.0/v1.8.1 approval files.

## Requirements
- [x] `SandboxExecutionApprovalStore.claim_for_execution()` validates all
      binding fields and atomically marks `consumed=True` using a lock file
      + `os.replace`.
- [x] `SandboxExecutionGate.execute_pending()` calls `claim_for_execution()`
      after preflight passes and BEFORE `ShellRunner.run()`.
- [x] Claim failure returns a blocked `SandboxExecutionResult` without
      calling the shell runner.
- [x] Stale lock files (> 60 s) are detected and cleaned up automatically.
- [x] `sandbox_execution_approval_claimed` audit event written on successful
      claim.
- [x] No OS sandbox binary is invoked.

## Backward Compatibility
- Preserve all existing CLI commands and their signatures.
- `consume()` method remains available for direct use.
- `is_approved()` behavior is unchanged (checks `consumed` field).
- Legacy approval files without `consumed` field are handled correctly.
- Existing test suites continue to pass.

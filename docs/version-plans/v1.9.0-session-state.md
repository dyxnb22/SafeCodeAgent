## Problem
SafeCode Agent has safe individual commands, but no durable interactive agent
session. Without session state, later agent-loop work has nowhere to store the
goal, plan, current step, pending action, observation, or recoverable failure.

## Goals
- [x] Add `.sac/session.json` as the current interactive agent session file.
- [x] Persist goal, plan, current step, pending action, last observation,
      status, last error, and timestamps.
- [x] Add `sac agent start`, `sac agent status`, and `sac agent clear`.
- [x] Write session state atomically through same-directory temp files and
      `os.replace`.
- [x] Invalid session JSON loads as no active session.

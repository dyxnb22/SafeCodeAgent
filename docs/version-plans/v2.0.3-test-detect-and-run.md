## Problem
The agent needs a first-class way to find the project's likely test command and
propose running it through the existing shell policy gate.

## Goals
- [x] Add `ProjectTestDetector` for common Python and Node test commands.
- [x] Detect `pytest -q` from pytest config, Python project metadata, or a
  `tests/` directory.
- [x] Detect `uv run pytest -q` when `uv.lock` is present.
- [x] Detect `npm test` or `pnpm test` from `package.json` test scripts.
- [x] Add shell command proposals that evaluate policy without executing.
- [x] Add `sac test detect`.
- [x] Add `sac test run` using the same policy, approval, audit, and runtime
  logging boundaries as controlled shell commands.
- [x] Prefer a policy-runnable detected candidate when multiple commands are
  available.

## Acceptance Criteria
- [x] Test detection covers pytest, uv, npm, and pnpm markers.
- [x] Detected commands are shown with policy proposal status before execution.
- [x] Running tests reuses `ShellRunner` and never bypasses command policy.
- [x] Test run attempts write audit events as `test_completed` or
  `test_blocked`.

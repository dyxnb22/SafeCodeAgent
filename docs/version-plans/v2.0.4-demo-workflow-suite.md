## Problem
SafeCode needs repeatable local demo tasks that exercise the user-facing MVP
workflows without relying on a particular developer's workspace state.

## Goals
- [x] Add a deterministic workflow suite for FastAPI, CLI, docs-only, and
      failing-test repair demos.
- [x] Each workflow includes seed files, a task prompt, expected touched files,
      workflow commands, and acceptance notes.
- [x] Add a CLI surface for listing, inspecting, and materializing demo
      projects.
- [x] Extend mock patch generation so each editable demo has a stable patch.
- [x] Add regression tests that verify the suite shape, materialization, test
      detection, mock patch coverage, and CLI commands.

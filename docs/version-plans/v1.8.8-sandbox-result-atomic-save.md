## Problem
Sandbox execution result records were written directly with `Path.write_text`.
That is simple, but it has two undesirable properties for a security-sensitive
result store:

- A process crash or interrupted write can leave a partial JSON record.
- An existing same-name symlink can be followed by direct writes.

## Goals
- [x] Write result records through a same-directory temporary file.
- [x] Atomically publish records with `os.replace(...)`.
- [x] Replace an existing same-name symlink rather than following it.
- [x] Clean up temporary files if the replace step fails.
- [x] Preserve the existing result record schema and load/list/filter behavior.

## Requirements
- [x] `SandboxExecutionResultStore.save(...)` writes to
      `.sac/sandbox_executions/.<target>.json.<uuid>.tmp`.
- [x] `save(...)` uses `os.replace(temp_path, target_path)`.
- [x] The `finally` block removes leftover temp files when possible.
- [x] Existing symlink targets are not modified by `save(...)`.
- [x] Failed replace operations do not leave `*.tmp` files in the result
      directory.

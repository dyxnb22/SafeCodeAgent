# v1.8.1: Single-Use Sandbox Execution Approval

## Status
Implemented.

## Base
- Depends on: `v1.8.0`
- Baseline context: `.claude/skills/current/SKILL.md`
- Runtime invariants: `.claude/skills/shared/core-runtime.md`
- New tag: `v1.8.1`

## Goals
- [x] Make sandbox execution approval single-use: after one successful
      execution the same approval cannot pass `is_approved()` again.
- [x] Blocked preflight (unsupported backend, command policy, network policy,
      etc.) must NOT consume the approval.
- [x] Audit must reflect approval consumption via a dedicated event
      `sandbox_execution_approval_consumed`.
- [x] Backward-compatible with approval files written by v1.8.0 that lack
      the `consumed` field.

## Requirements
- [x] `SandboxExecutionApproval` gains a `consumed: bool = False` field.
- [x] `SandboxExecutionApprovalStore.is_approved()` returns `False` when
      `consumed=True`.
- [x] `SandboxExecutionApprovalStore.consume()` marks an approval as consumed
      on disk, returning `False` if already consumed or missing.
- [x] `SandboxExecutionGate.execute_pending()` calls `consume()` after a
      successful `ShellRunner` execution.
- [x] Legacy approval files without `consumed` are treated as `consumed=False`.
- [x] No OS sandbox binary is invoked.

## Backward Compatibility
- Preserve all existing CLI commands and their signatures.
- Approval files from v1.8.0 without `consumed` field are handled correctly
  (`data.get("consumed", False)` in `_load()`).
- `REQUIRED_APPROVAL_FIELDS` is NOT extended with `consumed` to avoid
  rejecting legacy files.

## Problem
Approval-gated actions used different CLI wording and audit metadata. As the
interactive loop grows, users need a consistent checkpoint shape before writes
or execution.

## Goals
- [x] Add `HumanCheckpoint` and `HumanCheckpointPresenter`.
- [x] Write `human_checkpoint_presented` audit events.
- [x] Use consistent checkpoint panels for patch apply, medium-risk shell run,
      MCP write proposal, and sandbox execution approval.
- [x] Store a subject hash instead of raw subject text in checkpoint metadata.
- [x] Preserve all existing safety gates and execution behavior.

# v2.1.2: Runtime Consolidation

## Status
Implemented locally; tag pending.

## Base
- Depends on: `v2.1.1`
- New version: `v2.1.2`

## Problem
SafeCode has a strong safety runtime, but several product paths were still
fragmented: the CLI entrypoint carried every command group, the interactive
agent loop used a deterministic placeholder plan, context intelligence was not
fed into collection, and a few placeholder modules were ambiguous.

## Goals
- [x] Split `src/safecode/cli.py` into focused command modules without changing public commands.
- [x] Keep `from safecode.cli import app` and the `sac` console script compatible.
- [x] Connect `AgentLoop` to `LLMClient.plan()` and `LLMClient.choose_tool()`.
- [x] Include repo-map summaries and task-selected context in `ContextCollector.collect(query=...)`.
- [x] Keep context packing and source metadata in the collection path.
- [x] Replace or clarify placeholder utility surfaces.
- [x] Sync package version and version matrix paths.

## Testing Checklist

## Acceptance Criteria
- [x] CLI command groups are imported from focused modules.
- [x] Agent sessions receive structured LLM plans before stepping.
- [x] Each agent step routes the model-selected tool intent through `ToolIntentRouter`.
- [x] Context includes repository intelligence and selected file snippets when a query is supplied.
- [x] Version documentation describes the consolidation task.

# v2.1.4 — Context Debug Command

## Goal

Add `sac context explain "task"` so users can inspect why files and context snippets
were selected for a given task, without calling the LLM or writing any files.

## Motivation

Context selection is a black box to users today. When the agent selects wrong or
irrelevant files, there is no way to diagnose the issue without reading source code.
A read-only explain command surfaces the scoring, reasons, and budget constraints
that drive selection.

## Design

- New `src/safecode/cli_context.py` with `context_app` Typer group.
- Single command `context explain <task>` with `--root` and `--limit` options.
- Reuses `ContextSelector.select_sources()` for ranked file selection.
- Reads config budget limits from `SafeCodeConfig`; does not invoke `ContextBudgetPacker`.
- Reads repo-map statistics from `RepoMapBuilder.build()`.
- Output is deterministic and suitable for testing with `CliRunner`.
- Registered in `src/safecode/cli.py` under the `context` group.

## Constraints

- Read-only; no writes to `.sac/` or any other path.
- No LLM calls.
- Respects existing redaction and skip-path rules via `ContextSelector` → `FileIndexer`.
- Does not create a parallel context system; reuses existing types throughout.

## Acceptance

PYTHONPATH=src python3 -m pytest tests/test_context_budget.py tests/test_repo_map.py -q

# v2.2.2 Plan: MCP Read Tool Loop

## Goal
Allow the bounded agent loop to call approved read-only MCP tools through the existing registry/adapter/router safety boundary.

## Scope
- Add `MCPReadToolExecutor` in `src/safecode/mcp/loop_executor.py`.
- Update `ToolIntentRouter` to classify MCP tool names: read-only ‚Üí `mcp.call_readonly` (no approval); write/unknown ‚Üí `mcp.propose` (approval required).
- Update `AgentLoop.step()` to execute read-only MCP calls and capture observations.
- Add `record_mcp_call()` to `AgentJournalStore` with new `mcp_call` event type.
- Audit handled by existing `MCPReadOnlyRunner` audit path.
- Fail closed for unknown tools, non-read-only tools, bad input, missing config, runner errors.

## Hard Constraints
- No MCP writes (v2.2.3).
- No bypass of registry, adapter, router, audit, or approval boundaries.
- No shell execution through MCP.
- No weakening of network/config policy.
- Only tools classified as read-only by `classify_mcp_tool()` may be executed without approval.

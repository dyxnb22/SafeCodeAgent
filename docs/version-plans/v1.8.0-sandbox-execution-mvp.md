## Design: Why Noop-Only?

The Noop adapter runs commands through SafeCode's own `ShellRunner`, which
already enforces:

- **CommandPolicy** — high-risk commands blocked even with `--yes`.
- **NetworkPolicy** — network disabled by default.
- **FilesystemBoundary** — cwd is project root, no escape.

This is **local policy-gated execution**, not OS sandbox.  It exercises every
gate without introducing kernel-sandbox risk.  Native OS sandbox backends
remain dry-run only until a future version validates real sandbox execution
end-to-end.

## Problem
Sandbox approval files live outside the project root and gate real Noop
execution. The approval claim path already used `os.replace`, but approval
creation and the legacy consume helper still wrote JSON directly. The claim
path also used a fixed `.tmp` filename.

Direct writes are not ideal for approval state because interrupted writes can
leave partial JSON, and a same-name symlink can be followed by direct file
writes.

## Goals
- [x] Add one approval-store helper for atomic JSON writes.
- [x] Use same-directory random temp files plus `os.replace(...)`.
- [x] Use the helper for approval creation, legacy consume, and execution
      claim.
- [x] Replace same-name symlinks rather than writing through them.
- [x] Clean up leftover temp files if replace fails.

## Requirements
- [x] `SandboxExecutionApprovalStore._atomic_write_json(...)` writes hidden
      temp files under the approval directory.
- [x] Temp names include random UUID material, not a shared fixed `.tmp`.
- [x] `approve(...)`, `consume(...)`, and `claim_for_execution(...)` use the
      helper.
- [x] Existing binding checks, TTL checks, policy-version checks, lock-file
      behavior, and project-root trust boundary remain unchanged.

## Problem
Agent sessions persist current state, but users need a durable session timeline
that explains plan, action, failure, and final-summary events across commands.

## Goals
- [x] Add `AgentJournalStore` under the state layer.
- [x] Store per-session JSONL journals under `.sac/agent_journals/`.
- [x] Record plan events when sessions start or receive a default plan.
- [x] Record action events as the agent loop advances steps.
- [x] Record failure events when sessions are aborted.
- [x] Record final-summary events when sessions complete.
- [x] Expose `sac agent journal [session_id]`.
- [x] Include latest journal summary in Markdown task reports.
- [x] Provide journal methods for future diff and command events.

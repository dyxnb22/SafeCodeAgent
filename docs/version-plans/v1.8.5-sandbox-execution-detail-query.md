# v1.8.5: Sandbox Execution Detail + Query Polish

## Status
Planned.

## Base
- Depends on: `v1.8.4`
- Baseline context: `.claude/skills/current/SKILL.md`
- Runtime invariants: `.claude/skills/shared/core-runtime.md`
- New tag: `v1.8.5`

## Problem
v1.8.4 provides list/status filtering but no detailed single-record view
and no limit/sort controls on the query side. Looking up a specific
execution by proposal_id requires grepping through the full list or
reading the JSON file manually.

## Goals
- [x] Add `sac sandbox execution show <proposal-id>` CLI command for a
      detailed single-record view with all metadata and stdout/stderr
      previews.
- [x] Extend `filter_by()` with `limit` and `sort_order` parameters so
      filtering, sorting, and pagination are testable in one method.
- [x] Add `--limit N` and `--sort newest|oldest` to `sac sandbox executions`.
- [x] Validate `limit` (must be >= 1) and `sort_order` (newest/oldest)
      with clear ValueError.
- [x] Preserve all v1.8.4 semantics: schema version, robust reads,
      backward-compatible defaults.

## Requirements
- [x] `filter_by()` accepts `sort_order: str = "newest"` and
      `limit: int | None = None`. Raises `ValueError` on invalid values.
- [x] `sac sandbox execution show <proposal-id>`:
  - Shows full metadata table (proposal_id, attempted_at, backend,
    executed, exit_code, duration_ms, status, command_head,
    command_hash_prefix, message, stdout_length, stderr_length).
  - Shows stdout/stderr preview panels when present.
  - Returns non-zero exit code and error message when proposal_id not found.
- [x] `sac sandbox executions`:
  - `--limit N`: cap output to N records.
  - `--sort newest|oldest`: change sort order (default newest).
  - Invalid parameters caught and surfaced as errors.

## Backward Compatibility
- `filter_by()` defaults unchanged — `sort_order="newest"`, `limit=None`
  producing the same output as v1.8.4.
- `sac sandbox executions` without new flags produces identical output.
- All existing CLI commands and signatures preserved.

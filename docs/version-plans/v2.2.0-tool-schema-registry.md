# v2.2.0 — Tool Schema Registry

## Goal
Define structured, queryable schemas for every SafeCode internal tool so that future model tool calls (v2.2.1+) can be validated against a known registry before execution.

## Requirements

### Registry models (`src/safecode/tools/registry.py`)
- `ToolRiskLevel` StrEnum: `low`, `medium`, `high`.
- `PermissionCategory` StrEnum: `read`, `write`, `shell`, `sandbox`, `mcp`, `subagent`, `audit`.
- `ToolArgSchema` frozen Pydantic model: `name`, `type` (constrained literal), `required`, `description`.
- `AuditEventRef` frozen Pydantic model: `event_type`, `description`.
- `ToolSpec` frozen Pydantic model: `name`, `description`, `risk`, `permission_category`, `requires_human_approval`, `args`, `audit_event`.
- `ToolRegistry` class with `list()`, `get()`, `names()`, `by_permission()`, `by_risk()`, `requiring_approval()`.
- Registry is deterministic and keyless — no external state is read or written.

### Registered tools (16 total)
`audit.verify`, `checkpoint.rollback`, `context.collect`, `context.read`, `mcp.call_readonly`, `mcp.propose_write`, `patch.apply`, `patch.propose`, `report.render`, `sandbox.execute`, `sandbox.plan`, `sandbox.propose`, `shell.propose`, `shell.run`, `subagent.inspect`, `subagent.run_readonly`.

### CLI (`src/safecode/cli_project.py`)
- `sac tools list` — expanded to show Name, Risk, Permission, Approval, Description. Accepts `--risk` and `--permission` filters.
- `sac tools inspect TOOL_NAME` — rich panel with all fields; exits non-zero for unknown tool.

### Safety constraints
- Do not execute any tool from the registry.
- Do not call the LLM.
- Do not weaken shell, patch, approval, sandbox, audit, or redaction boundaries.
- High-risk tools must require human approval.
- WRITE and SHELL permission tools must require human approval.

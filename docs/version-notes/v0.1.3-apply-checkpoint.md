# v0.1.3 Apply + Checkpoint

## 实现内容

- `sac apply` 读取 pending patch。
- apply 前重新 validate。
- 再次展示 diff。
- 用户确认后创建 checkpoint。
- `PatchApplier` 真正写文件。
- 删除 `.sac/pending_patch.json`。
- 写入 `checkpoint_created` 和 `patch_applied` audit event。

## 入口

```bash
uv run sac apply
```

## 代码走向

```text
cli.py apply()
-> AgentOrchestrator.preview_apply()
-> PatchValidator.validate()
-> build_unified_diff()
-> typer.confirm()
-> AgentOrchestrator.apply()
-> CheckpointManager.create()
-> PatchApplier.apply()
-> AuditLogger.write()
```

## 验收命令

```bash
uv run sac edit "演示一次安全修改"
uv run sac apply
uv run pytest tests/test_patch_apply.py tests/test_checkpoint.py
```

## 学习重点

- 先 checkpoint，再写文件。
- apply 前不能相信 edit 时的校验结果，必须重新校验。
- 输入 `n` 时不能修改文件。

# v1.9.4 Release Notes — Human Checkpoint Prompts

## Summary

v1.9.4 adds standardized human checkpoints for approval-gated actions.

## What Changed

- Added `HumanCheckpoint`.
- Added `HumanCheckpointPresenter`.
- Added `human_checkpoint_presented` audit events.
- Connected checkpoints to:
  - patch apply
  - medium-risk shell run
  - MCP write proposal
  - sandbox execution approval

## Security

- Checkpoints do not approve actions by themselves.
- Checkpoint metadata stores a subject hash rather than the raw subject.
- Existing patch, shell, MCP, and sandbox safety gates remain unchanged.

## Backward Compatibility

- Existing CLI commands and flags are unchanged.
- The change adds checkpoint panels and audit events around existing flows.

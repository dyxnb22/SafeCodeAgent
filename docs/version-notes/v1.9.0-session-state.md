# v1.9.0 Release Notes — Agent Session State

## Summary

v1.9.0 starts the interactive agent-loop track by adding durable session state
under `.sac/session.json`.

## What Changed

- Added `AgentSessionState` and `AgentSessionStore`.
- Added `sac agent start "goal"`.
- Added `sac agent status`.
- Added `sac agent clear`.
- Session state records goal, plan, current step, pending action, last
  observation, status, last error, creation time, and update time.

## Security

- Session writes use same-directory temporary files plus `os.replace`.
- Existing same-name symlinks are replaced rather than followed.
- Invalid or corrupt session JSON loads as no active session.

## Backward Compatibility

- Existing `ask`, `edit`, `apply`, `run`, `sandbox`, MCP, and subagent commands
  are unchanged.

# v1.8.8 Release Notes — Sandbox Result Atomic Save

## Summary

v1.8.8 hardens sandbox execution result persistence. Result records are now
written through same-directory temporary files and published with
`os.replace(...)`.

## What Changed

### Atomic Result Save
- `SandboxExecutionResultStore.save(...)` writes JSON to a hidden temp file in
  `.sac/sandbox_executions/`.
- It then atomically replaces the final result path with `os.replace(...)`.
- Leftover temp files are cleaned up if publishing fails.

### Symlink Safety
- Saving a result over an existing same-name symlink replaces the symlink path
  itself.
- The symlink target is not modified.

## Security
- Reduces risk of partial result JSON after interrupted writes.
- Avoids direct writes through attacker-controlled same-name symlinks in the
  project-local result directory.
- Does not change result schema, redaction, truncation, prune behavior, or CLI
  output.

## Backward Compatibility
- Existing records still load normally.
- `SandboxExecutionResultStore.save(...)` still returns the saved record.
- `list_all()`, `filter_by()`, `latest()`, `stats()`, `plan_prune()`, and
  `prune()` keep the same public behavior.

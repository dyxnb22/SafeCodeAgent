# v2.1.1 Release Notes - Test Build Detector

## Summary

v2.1.1 expands project command detection beyond Python and Node test basics.

## What Changed

- Added Ruff lint command detection.
- Added npm/pnpm `build` and `lint` script detection.
- Added Gradle `test` and `build` detection with wrapper preference.
- Added Maven `test` and `package` detection.
- Added Go `test ./...` and `vet ./...` detection.
- Added Cargo `test` and `check` detection.
- Classified Gradle, Maven, Go, and Cargo as medium-risk developer commands.

## Security

- Detection still never executes commands.
- `sac test detect` displays policy status through `ShellRunner.propose`.
- `sac test run` still executes only through command policy, approval, runtime
  logging, and audit boundaries.
- Project-defined scripts such as `npm run lint` remain subject to existing
  command-policy checks.

## Backward Compatibility

- Existing pytest and npm/pnpm test detection order is preserved.
- Existing `sac test run` behavior is unchanged except that more candidates may
  be visible.

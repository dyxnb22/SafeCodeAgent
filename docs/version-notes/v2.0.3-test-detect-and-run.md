# v2.0.3 Release Notes - Test Detect and Run

## Summary

v2.0.3 adds project test detection and a policy-gated `sac test run` workflow.

## What Changed

- Added `ProjectTestDetector`.
- Added detection for pytest, uv pytest, npm test, and pnpm test commands.
- Added `ShellRunner.propose(...)` to evaluate commands through policy without
  executing them.
- Added `sac test detect` to list detected commands, confidence, and policy
  status.
- Added `sac test run` to run a detected or explicit command through the same
  shell policy, approval checkpoint, runtime log, and audit boundaries as
  `sac run`.
- Refined shell network checks so npm/pnpm network policy targets install-like
  operations while local `npm test`/`pnpm test` can still be evaluated by command
  policy.

## Security

- Detection never executes commands.
- Execution still goes through `ShellRunner`, `CommandPolicy`, and medium-risk
  approval checkpoints.
- Non-allowlisted commands remain blocked.
- High-risk commands remain blocked.
- Audit events record `test_completed` or `test_blocked`.

## Backward Compatibility

- Existing `sac run` behavior is unchanged.
- Existing network-deny tests for git, curl, npm install, pip install, and npx
  remain in force.

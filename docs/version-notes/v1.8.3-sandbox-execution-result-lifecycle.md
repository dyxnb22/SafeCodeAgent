# v1.8.3 Release Notes — Sandbox Execution Result Lifecycle

## Summary

v1.8.3 adds a persistent result record for every sandbox execution attempt
and ensures the pending proposal file is properly cleaned up after terminal
states (execution completed or approval claim failed).

## What Changed

### Result Records
- Each execution attempt (successful, non-zero exit, or claim-blocked) now
  writes a `SandboxExecutionResultRecord` to `.sac/sandbox_executions/<proposal_id>.json`.
- Records include: proposal_id, attempted_at, backend, executed, exit_code,
  duration_ms, status, message, command_hash_prefix, command_head, truncated
  stdout/stderr preview, and output lengths.
- stdout/stderr are truncated to 2000 characters. Full command argv is never
  stored — only a hash prefix and the first token.

### Pending Proposal Lifecycle
- **Execution completed (any exit code)**: result record written, pending
  proposal cleared.
- **Claim failure** (approval already consumed, expired, or mismatched):
  `blocked_claim` result record written, pending proposal cleared.
- **Preflight blocked**: pending proposal preserved so the user can fix
  configuration and retry. No result record written because no execution
  was attempted.

### New CLI Commands
- `sac sandbox executions` — lists all past execution result records
  (newest first).
- `sac sandbox last-execution` — shows the most recent result record with
  metadata table and stdout/stderr previews.

## Security
- Result records never include env keys or env values in structured fields.
- Full command argv is not stored — only `command_hash_prefix` (first 16 chars
  of SHA-256) and `command_head` (first token).
- stdout/stderr are truncated to `MAX_RESULT_PREVIEW_LENGTH` (2000 chars) to
  prevent disk bloat and limit sensitive content exposure.
- v1.8.2 safety semantics are fully preserved: atomic claim before execution,
  blocked preflight does not consume approval.

## Backward Compatibility
- All existing CLI commands and signatures unchanged.
- Preflight, approval, claim, and audit behavior preserved.
- 3 existing tests updated to account for pending-clearing-after-execution.

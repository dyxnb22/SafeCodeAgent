# v2.1.3: Diff Planner Notes

## Summary

This release adds a lightweight diff planner that predicts which files a task
is likely to touch before the LLM generates a patch, then compares that
prediction to the actual patch scope and warns when they diverge.

## Implementation Notes

- `src/safecode/agent/planner.py` is a new standalone module. `DiffPlanner`
  has two methods: `predict(task, context_hint="")` and `compare(plan, proposal)`.
- `predict()` extracts file-path-like tokens from the task description using a
  regex that requires a word boundary and rejects URL path components (`:`, `/`,
  and `.` immediately before the match start). No LLM call is made.
- `compare()` performs exact POSIX path matching between `DiffPlan.predicted_files`
  and the files in `PatchProposal.blocks`. It returns a `DiffScopeResult` with
  one of four statuses: `no_prediction`, `match`, `within_scope`, `extra_files`.
  A warning string is populated only on `extra_files`.
- `EditResult` (in `orchestrator.py`) gains an optional `scope_result` field.
  The `edit()` method calls `predict()` before the LLM and `compare()` after
  parsing the patch. The `patch_proposed` audit event now includes
  `scope_status` and, if applicable, `scope_warning` in its metadata.
- `cli_core.py` prints the scope warning in yellow after the diff preview.
- The scope check is advisory: it never blocks or modifies a valid patch.

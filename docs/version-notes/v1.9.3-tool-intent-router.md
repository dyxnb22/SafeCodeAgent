# v1.9.3 Release Notes — Tool Intent Router

## Summary

v1.9.3 adds typed tool intents for the interactive agent loop. This creates a
safe validation boundary before later model-driven tool calls are connected to
real runtime actions.

## What Changed

- Added `ToolIntent`.
- Added `RoutedToolIntent`.
- Added `ToolIntentRouter`.
- Supported intent categories:
  - `read`
  - `patch`
  - `shell`
  - `sandbox`
  - `mcp`
  - `subagent`
  - `report`
- `AgentLoop.step(...)` now records routed read intent metadata in the session
  pending action.

## Security

- Unknown intent types fail closed.
- Missing required fields fail closed.
- Patch, shell, sandbox, and MCP intents are marked approval-required.
- The router validates and routes only; it does not execute tools.

## Backward Compatibility

- Existing `sac agent start/status/clear/step/run` commands remain available.
- Session JSON gains route metadata in `pending_action`.

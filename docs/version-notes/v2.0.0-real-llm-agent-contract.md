# v2.0.0 Release Notes - Real LLM Agent Contract

## Summary

v2.0.0 starts the usable local coding agent MVP line by defining a structured
contract for real LLM planning and tool selection.

## What Changed

- Added validated response schemas for `answer`, `plan`, `tool_intent`, `patch`,
  `stop_for_user`, and `error`.
- Added `parse_agent_contract_response(...)` for JSON and fenced JSON outputs.
- Extended `LLMClient` with `plan(...)` and `choose_tool(...)`.
- Added deterministic mock responses for plan and tool-choice flows.
- Added OpenAI-compatible structured plan/tool-choice calls.
- Added tests for contract parsing, fail-closed validation, mock behavior, and
  OpenAI-compatible structured method parsing without live network access.

## API Key Boundary

- Normal tests and mock-mode development do not require a real API key.
- Manual real-model validation starts here. Real LLM mode requires
  `OPENAI_API_KEY` or `SAFECODE_LLM_API_KEY`, plus the existing network policy
  allowing the configured LLM endpoint.

## Security

- Unknown response types fail closed.
- Tool intent payloads reuse the existing `ToolIntent` schema and router.
- The real-model client parses structured JSON before returning plan/tool
  decisions to the runtime.
- Write and execute tool categories still require the existing approval gates.

## Backward Compatibility

- Existing `ask(...)` and `propose_patch(...)` behavior is preserved.
- Mock mode remains the default for local tests.

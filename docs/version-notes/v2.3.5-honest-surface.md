# v2.3.5 — Honest Surface

## Summary

Makes SafeCode Agent's CLI output and docs accurately communicate current enforcement boundaries before real sandbox backends are added in v2.4.x.

## Changes

### Sandbox honesty (`src/safecode/cli_sandbox.py`)

- `sac sandbox status` now opens with a clear **Execution Scope (v2.3.x)** panel stating that v2.3.x executes only through the Noop backend and that macOS Seatbelt, Linux Bubblewrap, and Docker are plan-only / dry-run.
- The capability table gains a **Mode** column: `executing` for Noop, `plan-only` for all other backends.
- `sac sandbox plan` adds a **Backend Mode** row immediately after the Backend row in the plan table, so the plan-only status is visible in the table rather than hidden in a trailing note.
- The trailing "Dry Run" panel now references v2.3.x and names the non-Noop backends explicitly.

### MCP honesty (`src/safecode/cli_mcp.py`)

- `sac mcp --help` now clearly states that current MCP support is a subprocess JSON shim, not a full MCP JSON-RPC client (no live `tools/list` or `tools/call` protocol).
- `sac mcp tools` output includes a dim footer note restating the shim nature.
- The `mcp tools` command docstring clarifies that tools come from `.sac/mcp.toml` config, not a live JSON-RPC call.

### Subagent honesty (`src/safecode/cli_subagent.py`)

- `sac subagent --help` now describes subagents as read-only context/result collectors that collect project context and write a result file, not independent LLM investigations.
- `sac subagent run-readonly --help` repeats this characterization in the command docstring.
- The run-readonly result panel includes a dim note that no independent LLM investigation was performed.

### Tests (`tests/test_cli_output_honesty.py`)

21 new focused tests covering:
- `sandbox status`: v2.3.x scope notice present; non-Noop backends labeled plan-only; Noop identified as executing.
- `sandbox plan`: plan-only wording in table (not only in trailing note); Backend Mode row present; trailing note references v2.3.x not v1.7.x.
- `mcp tools` and `mcp --help`: subprocess/shim wording present.
- `subagent --help` and `subagent run-readonly --help`: collector / read-only wording present.

### Docs

- `docs/install-update.md` already contained the correct enforcement boundary summary from v2.3.3; no change required.
- `docs/version_implementation_matrix.md` already contained the v2.3.5 planned entry; confirmed accurate.

## What was NOT changed

- No real sandbox execution was added.
- No real MCP JSON-RPC client was implemented.
- No LLM-driven subagents were implemented.
- No public command names were renamed.
- All safety gates (diff review, checkpoint, audit, rollback, policy) are unchanged.

## Compatibility

- All existing command names and public APIs are unchanged.
- The `SandboxBackend` enum import is added to `cli_sandbox.py` for the mode comparison.
- `985 / 985` tests pass.

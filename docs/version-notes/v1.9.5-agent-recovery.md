# v1.9.5 Release Notes — Agent Recovery

## Summary

v1.9.5 adds recovery controls for interactive agent sessions.

## What Changed

- Added `AgentSessionStore.abort(...)`.
- Added `AgentSessionStore.resume(...)`.
- Added `AgentSessionStore.explain_last_failure(...)`.
- Added `sac agent abort --reason`.
- Added `sac agent resume`.
- Added `sac agent explain-last-failure`.

## Security

- Recovery commands only mutate session state.
- Completed sessions cannot be resumed.
- Missing sessions fail closed with a non-zero exit code for resume/abort
  operations.

## Backward Compatibility

- Existing `sac agent start/status/clear/step/run` behavior is unchanged.

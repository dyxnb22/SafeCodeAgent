# v2.0.5 Release Notes - MVP Docs

## Summary

v2.0.5 adds a guided MVP onboarding path for installing SafeCode, configuring
model mode, running the first realistic task, understanding safety boundaries,
and rolling back.

## What Changed

- Added `docs/mvp-user-guide.md`.
- Linked the MVP guide from README install, model, and first demo sections.
- Documented real LLM configuration with user-level and project-level network
  opt-in.
- Anchored the first task on the repeatable `failing-test-repair` demo workflow.
- Added documentation regression coverage for the guide and README links.

## Security

- Documentation keeps the default `mock` provider as the recommended demo mode.
- Real LLM instructions preserve the current safety model: project config cannot
  force provider changes or enable network access alone.
- The first task preserves edit-before-apply, checkpoint, audit, policy-gated
  tests, and rollback expectations.

## Backward Compatibility

- No runtime behavior changed.
- Existing commands and demo workflows are unchanged.

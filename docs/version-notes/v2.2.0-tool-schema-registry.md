# v2.2.0 — Tool Schema Registry (completion notes)

## What changed

### `src/safecode/tools/registry.py` (rewritten)
Replaced the minimal `ToolSpec` dataclass and `ToolRegistry` stub with a complete schema layer:

- **`ToolRiskLevel`** (`StrEnum`): `low` / `medium` / `high`.
- **`PermissionCategory`** (`StrEnum`): `read` / `write` / `shell` / `sandbox` / `mcp` / `subagent` / `audit`.
- **`ToolArgSchema`** (frozen Pydantic model): each argument has `name`, `type` (constrained `Literal`), `required`, `description`.
- **`AuditEventRef`** (frozen Pydantic model): `event_type` + `description` pointer to the audit log event a tool emits.
- **`ToolSpec`** (frozen Pydantic model): full tool metadata — name, description, risk, permission_category, requires_human_approval, args list, audit_event.
- **`ToolRegistry`**: `list()` (sorted), `get(name)` (raises `KeyError` for unknown tools), `names()`, `by_permission()`, `by_risk()`, `requiring_approval()`.

16 internal tools registered: `audit.verify`, `checkpoint.rollback`, `context.collect`, `context.read`, `mcp.call_readonly`, `mcp.propose_write`, `patch.apply`, `patch.propose`, `report.render`, `sandbox.execute`, `sandbox.plan`, `sandbox.propose`, `shell.propose`, `shell.run`, `subagent.inspect`, `subagent.run_readonly`.

### `src/safecode/cli_project.py`
- `sac tools list`: expanded table (Name, Risk, Permission, Approval, Description); new `--risk` and `--permission` filters.
- `sac tools inspect TOOL_NAME`: new command — rich panel showing all fields including arguments and audit event. Exits non-zero for unknown tool names.

### `tests/test_tool_schema_registry.py` (new — 40 tests)
Covers: model immutability, serialization round-trips, registry lookup, missing-tool errors, risk/permission metadata invariants (high-risk → approval required, WRITE/SHELL → approval required), audit event presence, query helpers, no-side-effects, and CLI smoke tests.

## Safety invariants preserved
- No tool is executed by the registry layer.
- No LLM is called.
- Shell, patch, approval, sandbox, audit, and redaction boundaries are unchanged.
- High-risk and WRITE/SHELL-permission tools all carry `requires_human_approval=True` (enforced by tests).

# v1.9.1 Release Notes — Agent Step Command

## Summary

v1.9.1 adds the first interactive loop primitive: `sac agent step`.

## What Changed

- Added `AgentLoop`.
- Added `sac agent step "goal"`.
- A missing session plus a supplied goal creates a new session.
- Each invocation advances exactly one plan item.
- The step records a read-only pending action and observation.

## Security

- The step command does not apply patches, run shell commands, call MCP tools,
  or execute sandbox proposals.
- Calling `sac agent step` without an existing session or goal fails closed.

## Backward Compatibility

- Existing `sac agent start/status/clear` behavior is unchanged.

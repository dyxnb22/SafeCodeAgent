# v2.0.1 Release Notes - Context Budget Manager

## Summary

v2.0.1 adds explicit context budget metadata so the agent can explain what it
sent to the model and what was truncated.

## What Changed

- Added `ContextBudget`, `ContextBudgetPacker`, `ContextBudgetReport`, and
  `ContextSource`.
- `ContextCollector.collect()` now preserves legacy context fields and adds a
  `context_budget` metadata block.
- Context budget metadata includes `max_bytes`, `max_tokens`, `bytes_used`,
  `tokens_estimated`, packed `sources`, and `truncation_notes`.
- Text truncation now respects UTF-8 byte boundaries.
- `ContextSelector` now exposes `select_sources(...)` with score and reason
  metadata while preserving the existing `select(...)` path-list API.

## Security

- Existing secret file skipping, symlink rejection, project-root redaction, and
  content redaction remain in place.
- The budget report records source names and size metadata, not hidden file
  contents.
- The collector still respects the safer merged `max_context_chars` value.

## Backward Compatibility

- Existing callers can continue reading `project_root`, `files`, `readme`, and
  `pyproject`.
- `ContextSelector.select(...)` still returns `list[str]`.

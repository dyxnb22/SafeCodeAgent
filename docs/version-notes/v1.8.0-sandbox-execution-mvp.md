# v1.8.0: Local Policy-Gated Execution MVP

## Summary

Enables real command execution through the **Noop sandbox adapter** when all
preflight checks pass. The command runs through SafeCode's existing logical
boundaries (CommandPolicy, NetworkPolicy, FilesystemBoundary) — this is
**local policy-gated execution**, not OS-level sandbox isolation.

macOS Seatbelt, Linux Bubblewrap, and Docker adapters remain dry-run only.

## Changes

### Source
- `src/safecode/sandbox/adapter.py` — `NoopSandboxAdapter.supports_execution()`
  returns `True`. All other adapters still return `False`.
- `src/safecode/sandbox/execution.py` — `execute_pending()` runs preflight
  checks, then executes via `ShellRunner` when allowed. Audit events use the
  unified `sandbox_execution_completed` / `sandbox_execution_blocked` types.
- `src/safecode/cli.py` — `sac sandbox execute` displays real exit code,
  stdout, stderr on success; failure reasons on block.

### Tests
- `tests/test_sandbox_execution_security_evals.py` — 30 new security evals
  covering execution gate, blocked paths, audit semantics, and regression.
- Updated 10 existing test files to reflect Noop execution support.

### Safety Invariants
All preserved from v1.7.9:
- Proposal integrity (hash verification)
- Approval binding (user-level, outside project root, TTL enforced)
- Command policy (dangerous commands blocked even with approval)
- Network policy (disabled-by-default enforced)
- Filesystem boundary (no escape from project root)
- Audit (every execution attempt logged)
- No OS sandbox binary invoked

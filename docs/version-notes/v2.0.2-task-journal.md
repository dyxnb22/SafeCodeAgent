# v2.0.2 Release Notes - Task Journal

## Summary

v2.0.2 adds durable per-session task journals for the interactive agent loop.

## What Changed

- Added `AgentJournalEvent`, `AgentJournalSummary`, and `AgentJournalStore`.
- Agent journals are stored as JSONL under `.sac/agent_journals/`.
- Session start/default-plan paths record `plan` events.
- Agent loop steps record `action` events.
- Session abort records `failure` events.
- Completed sessions record `final_summary` events.
- Added reusable `record_diff(...)` and `record_command(...)` methods for the
  next workflow layers.
- Added `sac agent journal [session_id]` to render a journal as Markdown.
- Markdown reports now include a compact latest-journal summary.

## Security

- Journal session ids are validated before building file paths.
- Journal writes use same-directory temporary files and `os.replace(...)`.
- If a journal target path is a symlink, saving replaces the symlink path rather
  than following it.
- Journal entries store structured summaries, not hidden context contents.

## Backward Compatibility

- Existing session JSON format remains unchanged.
- Existing `sac agent start/status/clear/abort/resume/step/run` behavior is
  preserved, with journal files added as side effects.
- Existing report output still starts with `# SafeCode Task Report`.
